Task 1: Drawing Single-Color Triangles

To rasterize a triangle, I implemented a point-in-triangle test using the edge equations. First, I determine the bounding box of the triangle by finding the minimum and maximum X and Y coordinates among the three vertices. I clamp these bounds to the edges of the screen to avoid out-of-bounds memory accesses. Next, I iterate through every integer pixel coordinate within this bounding box. For each pixel, I calculate the center point at (x + 0.5, y + 0.5). I then evaluate the three edge functions (\(L_0, L_1, L_2\)) by taking the 2D cross product of the edge vectors and the vector from the vertex to the sample point. If the sample point evaluates to \(\ge 0\) for all three edge functions (after ensuring consistent counter-clockwise winding order), the center point is inside the triangle, and I fill the pixel with the given color.

Algorithm Efficiency: My algorithm evaluates the exact bounding box of the triangle, meaning it is no worse than one that checks each sample within the bounding box. By calculating the mathematical floor of the minimum coordinate and the ceiling of the maximum coordinate, the nested for loops explicitly limit their bounds to the smallest possible rectangular grid containing the triangle.

Task 2: Antialiasing by Supersampling

Algorithm and Data Structures: Supersampling is a spatial anti-aliasing technique used to reduce "jaggies" (aliased pixelated edges). Instead of taking one boolean sample at the center of a pixel, I sample the pixel at a higher frequency (e.g., 4 or 16 times per pixel) and average the results. To support this, I updated the sample_buffer data structure, dynamically resizing a 1D vector to hold width * height * sample_rate colors. This effectively acts as a high-resolution virtual framebuffer.

Pipeline Modifications: In rasterize_triangle, I added two inner loops to iterate sqrt(sample_rate) times across the X and Y axes of each pixel. I calculate the sub-pixel sample points and check the edge equations for each sub-pixel, storing the color in the expanded sample_buffer. Finally, in resolve_to_framebuffer, I iterate over every standard display pixel, fetch the sample_rate corresponding colors from the sample_buffer, and average them together before writing the final 8-bit RGB color to the display target.

Task 4: Barycentric coordinates

Barycentric coordinates (\(\alpha, \beta, \gamma\)) are a local coordinate system for a triangle where any point inside the triangle can be expressed as a weighted sum of its three vertices.

The equation is:

$$ V = \alpha V_0 + \beta V_1 + \gamma V_2 $$

Where \(\alpha + \beta + \gamma = 1\). You can think of these coordinates as "proximity weights"—the closer a point is to a specific vertex, the higher that vertex's weight will be. Because these weights transition linearly across the surface of the triangle, we can use them to smoothly interpolate values assigned to the vertices. Task 5: "Pixel sampling" for texture mapping

Pixel sampling determines how we fetch a color from a texture image given a continuous floating-point texture coordinate (u, v). Because screen pixels rarely map exactly 1-to-1 with texture pixels (texels), we need a rule for what color to pick.

  • Nearest Sampling: I scale the (u,v) coordinates by the texture dimensions and round to the nearest integer index. It simply grabs the single closest texel. It is computationally cheap but results in blocky, pixelated images when zoomed in.
  • Bilinear Sampling: I find the four closest texels surrounding the exact floating point (u,v) location. I calculate the fractional offset (s, t) and perform three linear interpolations: two horizontally between the top/bottom texel pairs, and one vertically between the results.

Relative Differences: There will be a massive difference between the two methods when under magnification (when you zoom in closely to the texture). When zoomed in, nearest sampling will map multiple screen pixels to the exact same texel, creating large harsh square blocks. Bilinear sampling prevents this by smoothing the transition between adjacent texels, creating smooth gradients instead of blocks. Bilinear also helps smooth out high-frequency details that might otherwise "pop" or flicker when using nearest sampling.

Task 6: "Level Sampling" with mipmaps for texture mapping

Level sampling (mipmapping) involves sampling from pre-computed, lower-resolution versions of a texture based on how far away the textured object is. This is crucial for minification; if a 1000x1000 texture is squeezed into a 10x10 pixel area on screen, sampling the full-res texture will cause severe aliasing and moire patterns because adjacent screen pixels will jump wildly across the texture space.

Implementation: Inside rasterize_textured_triangle, I calculated the barycentric coordinates not just for (x, y), but also for (x+1, y) and (x, y+1). Using these, I found the corresponding (u,v) coordinates for the neighboring pixels, allowing me to calculate the difference vectors \(\frac{du}{dx}\), \(\frac{dv}{dx}\), \(\frac{du}{dy}\), and \(\frac{dv}{dy}\). In get_level, I scaled these derivatives by the texture dimensions and computed the maximum rate of change \(L\). Finally, the mipmap level was calculated as \(D = \log_2(L)\). Depending on the level sample method, I either clamped to level 0, rounded to the nearest integer level, or performed an additional linear interpolation between the two adjacent integer levels.

Tradeoffs:

  • Pixel Sampling: Fast, uses no extra memory, but has very weak antialiasing power for distant textures (highly susceptible to moire patterns).
  • Level Sampling (Mipmapping): Slightly slower (requires computing derivatives) and uses ~33% more memory to store the mipmap levels. However, it provides excellent antialiasing for minified textures and improves memory cache coherency.
  • Supersampling: The slowest method (rasterization math scales directly with the sample rate) and most memory intensive (requires \(N\) times more framebuffer memory). However, it provides the absolute best antialiasing power, especially for geometric edges where texture sampling cannot help.
