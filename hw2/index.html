Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm is a recursive method to evaluate Bezier curves. Given a set of n points and a parameter t (0<=t<=1), the algorithm computes n-1 intermediate points by linearly interpolating each adjacent control point pair.

In the evaluateStep function, we loop through the given vector of points and apply the standard linear interpolation formula to each adjacent pair:
$$P_i' = (1 - t)P_i + tP_{i+1}$$
These new points are stored in a new vector and returned. This is one step of the algorithm which can be recursed on.


Part 2: Bezier surfaces with separable 1D de Casteljau

The 1D version of the de Casteljau algorithm also works on 2D Bezier surfaces by evaluating the surface as a grid of 1D Bezier curves. From a 2D grid of control points (n x m) and two parameters (u, v), the surface is evaluated in two passes.

The evaluate function iterates through each of the n rows of control points. Each row is treated as its own 1D curve and evaluated at parameter u using evaluate1D as a helper function. This reduces the 2D grid to a single column of evaluated points. Then this last vector is evaluated at v to find the 3D point on the surface.



Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To implement smooth shading, I calculate the area weighted normal at a given vertex by averaging the normals of all its neighboring faces that aren't on a boundary.

We can iterate through half-edges originating from the central vertex. At each triangle face I take the coordinates of its three vertices. I computed two edge vectors by subtracting the central vertex position from the other two vertices. Taking the cross product of these two vectors gives a vector orthogonal to the face. The magnitude of a cross product is proportional to the area of the parallelogram formed by the two vectors, so adding these products together weights them by the area of the respective triangles. The weight is then normalized with a call to .unit().

Part 6: Loop subdivision for mesh upsampling

I implemented Loop subdivision by breaking it down into the five passes over the mesh specified in the file:

  1. Compute new positions for original vertices: I iterated through all original vertices. Using the Loop subdivision weighting rule, I computed the new positions based on the degree and the sum of neighboring vertex positions. These were then stored in Vertex::newPosition and marked isNew = false.
  2. Compute positions for new vertices: I iterated over all the edges again and calculated the position of the vertex that would be created when it was split. I stored this in Edge::newPosition.
  3. Split original edges: I iterated through a isolated list of original edges and called splitEdge on each. I assigned the new vertex position to the value I stored on the edge in Step 2, and marked the vertex as isNew = true.
  4. Flip new edges: I then iterated through all of the edges. If an edge was marked isNew and it was connected to one old vertex and one new vertex, I flipped it to correct the topology.
  5. Update positions: Lastly, I iterated through all the vertices and updated their position data with the values stored in newPosition.

Observations: With more subdivisions, sharp edges and corners become smooth, which ends up shrinking overall volume. To reduce this effect you can split edges near sharp features which adds more geometry near a corner. This makes the area-weighted averaging have less influence over the corner vertex, preserving the features better.

Cube Asymmetry: When subdividing cube.dae, it becomes slightly asymmetric. The original cube mesh consists of exactly two triangles per face, meaning each square face is split by a single diagonal edge. Because Loop subdivision uses the topological degree of vertices, vertices that have the diagonal attached have a different degree and are weighted differently that vertices without. This causes the shape to subdivide asymmetrically.

The Fix: By splitting the single diagonal on each face so that every vertice has a diagonal connected to it, the corners of the cube have the same topological degree. When subdivided, the weighting is symmetrical.
